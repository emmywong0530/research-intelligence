The intended contract is: API keys, installation secrets, and companion-owned
approval codes are never returned; `pairing/complete` intentionally returns one
short-lived bearer session token. The frontend keeps that token only in memory;
it is not written to localStorage, sessionStorage, workspace files, logs, source
control, or packaged artifacts, and it expires or is invalidated on companion
restart. This is a checkpoint wording clarification, consistent with
`docs/local-api.md` and the implementation, not a product implementation
blocker or a reason to redesign pairing.
