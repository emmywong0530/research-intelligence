## Merge blockers

1. **Security contract clarification/blocker:** the current pairing contract
   returns a short-lived `session_token` from `pairing/complete`, while the
   checkpoint wording asks for no session secrets in API responses. Resolve this
   contradiction in a security/API decision before claiming the stronger
   no-secret-in-response requirement. Recommended follow-up: either explicitly
   define the returned token as a one-time in-memory bearer credential or
   redesign the exchange using a browser-safe mechanism.
2. **Full PWA integration evidence:** browser launch aborts locally, and no
   successful CI run is recorded in the repository. Do not claim end-to-end
   verified or production-ready PWA/companion pairing until a browser run passes.
